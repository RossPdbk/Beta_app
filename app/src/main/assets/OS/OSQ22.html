Q22. Explain in detail : File management in Linux.

  • Linux includes versatile and a powerful file handling facility, designed to support a wide variety of file management system and file structures.

  • Linux kernel handles all various types of files by hiding complex implementation behind a layer of software called as "Virtual File System" (VFS).

  • The VFS defines a common file model that capable of representing file system features and behaviour.

  • Here, VFS assumes that the files are objects in computer memory. The hardware specification or target file system has no effect on it.

  • VFS defines four object types. They are as follows :
    1. An inode object --> Repesents an individual file.
    2. A dentry object --> Represents an individual directory entry.
    3. A superblock object --> Represents an entire file system.
    4. A file object --> Represents an open file.

  • Every object of above file has pointer associated with it which points to function table.

  • Function table contains address of functions that actually implements the process.

  • There is set of operations prescribe for each of four object types.

  • The VFS software is able to perform an operation on file system objects by calling appropriate function from object function table without having to know what kind of object user is dealing with.


Examples of file management commands in linux.

  1. cd --> Change current directory
  2. dir --> List directory contents
  3. ls --> List files present in that directory
  4. pwd --> Print working directory with full path
  5. cmp --> Compare two files
  6. echo --> Display lines in text file
  7. mkdir --> Create a new folder
  8. rmdir --> Remove an existing directory
