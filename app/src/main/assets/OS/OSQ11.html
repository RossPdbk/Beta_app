Q11. Write difference between Pre-emptive and Non-preemptive scheduling.

No Preemptive Scheduling Non-Preemptive Scheduling
1. Processor can be preempted to execute a different process in the middle of execution of any current process. Once Processor starts to execute a process it must finish it before executing the other. It cannot be paused in middle.
2. CPU utilization is more compared to Non-Preemptive Scheduling. CPU utilization is less compared to Preemptive Scheduling.
3. Waiting time and Response time is less. Waiting time and Response time is more.
4. The preemptive scheduling is prioritized. The highest priority process should always be the process that is currently utilized. When a process enters the state of running, the state of that process is not deleted from the scheduler until it finishes its service time.
5. If a high priority process frequently arrives in the ready queue, low priority process may starve. If a process with long burst time is running CPU, then another process with less CPU burst time may starve.
6. Preemptive scheduling is flexible. Non-preemptive scheduling is rigid.
7. Ex:- SRTF, Priority, Round Robin, etc. Ex:- FCFS, SJF, Priority, etc
