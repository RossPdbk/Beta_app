Q21. Protection Mechanism

  • Memory protection in a paged environment is accomplished by protection bits associated with each frame.

  • Suppose we have read-only file. Every reference to the memory goes through page table to find correct frame.

  • At the same time, Physical address is being computed. The protection bit can be checked to verify that no-write are being made.

  • For this finer level protection, we can create a hardware to provide read-only, read-write or execute only protection by providing seperate protection bits.

  • This bit is valid/invalid bit. When bit is set to 'Valid', Page is legal and address space is sent and if bit is set to 'Invalid' then page is not allowed.

  • E.g. The system with 14 bit address space (0-16383), But we have a program that has address only from 0 to 10468. so address in page 0, 1, 2, 3, 4, 5 are mapped normally as they have valid bit but page 6 & 7 are not allowed as they are attached with invalid bit.
