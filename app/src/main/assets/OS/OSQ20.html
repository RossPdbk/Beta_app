Q20. Explain hardware support for paging.

  • The hardware implementation of page table can be done in several ways. In simplest case, Page table is implemented as set of dedicated registers. But the problem with this approach is the time required to access a user memory location. This problem is overcome by TLB.

  • The translation look-aside buffer is a special, small, fast look-up, hardware cache.

  • The TLB is associative, high speed memory.

  • Each entry in TLb has two parts : 1. Key (tag), 2. Value

  • When associative memory is present with an item, The item is compared with all keys simultaneously. If item is found corresponding value field is return.

  • The search is fast but hardware is expensive.

  • The number of entries in TLB is small often numbering between 64 to 1024.

  • The TLB is usedwith the page table in the following way :
    1. The TLB contains only few of page table entries when logical address is generated by CPU. Its page number is represented to TLB.
    2. If page number is found, Its frame number is immidiately available and ready to access memory
    3. If page number is not found in TLB, A memory reference to the page table must be made where frame number is obtained and is ready to access memory.

  • The percentage of times that a particular page number is found in the TLB is called as Hit ratio.

  • Effective access time = (t+E)a+(2t+E)(1-a) = (2-a)t+E
    Here, a = hit ratio, E = assciative look-up
