Q26. System calls

  • System calls provides an interface to the service made available by OS.

  • Application developers often do not have direct access to system calls, but can access them through an application programming interface (API).

  • When a programmer in user mode requires access to the RAM or hardware resource, It must ask the kernel to provide access to that resources. This is done via something called system calls.

  • When a programmer makes a system call, The mode is switched from user to kernel mode. this is called as context switch.

  • The a kernel provides resource which the program requested. After that another context switch happens which results in change of mode from kernel back to user mode.

System call parameters
Three methods to pass parameters :

  1. Parameters can be passed in registers.

  2. When there are more parameters than registers, parameter can be stored in block & the block address can be pass as parameter.

  3. Parameters can also be pushed or popped

Types of System calls :

  1. Process control
    Functions :
    • Eat, abort
    • load, execute
    • Create process, terminate process.
    • Wait for time
    • Wait event, Signal event
    • get/set process attribute

  2. File manipulation
    Functions :
    • Create file, delete file.
    • Open file, Close file.
    • read, write, reposition

  3. Device management
    Functions :
    • Request deviec, release device
    • Get/Set device attribute
    • read, write, reposition

  4. Information management
    Functions :
    • Get/Set time and date.
    • Get/Set system data
    • Get/Set process, file or device attribute

  5. Communication
    Functions :
    • Create, Delete communication connection
    • Send/Receive message
