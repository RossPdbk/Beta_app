Q3. What are different page replacement policies?

1. FIFO algorithm

  • The simplest page replacement algorithm and work on the basis of first in first out. It throws out the pages in the order in which they were brought in.

  • This algorithm always chooses the oldest page for replacement.

  • Since the replacement is FIFO, a queue can be maintained to hold all the pages in main memory.

  • The time is associated with each page when it is brought into the main memory.

  • This algorithm does not care about which pages are accessed frequently and which are not.

2. Optimal page replacement

  • This algorithm replaces the page which will not be used for longest perios of time.

  • This algorithm checks whether the page will be used in future or not.

  • As page repacement are done on basis of future uses, it is impossible to implement as it is impossible to determine number of instructions that will be executed in future.

  • Optimal page replacement algorithm has lowest page fault rate among all algorithms.

3. LRU page replacement

  • LRU - Least Recently used.

  • The time of page's last use is associated with each page.

  • LRU first determines which pages were recently used and then chooses that page that was used farthest back in the past.

  • This algorithm replaces page on basis of past use of the page which is possible as we can determine how many pages were used in the past.

  • This algorithm is consider to be one of the best page replacement algorithm.

  • this algorithm does not suffer through Belady's anamoly.

4. NFU page replacement

  • NFU - Not Frequently Used.

  • This policy assumes that the pages which are used more frequently will be needed again in the future.

  • A counter is used to keep track of number of times a page was used.

  • The page which has less ccounter value should be replaced.

  • the main drawback of this policy is if initially page is used frequently and then there is no reference again to this page, it will still remain in memory.

5. Most frequently used page replacement

  • This algorithm works opposite of NFU algorithm.

  • This policy assumes that page which is least frequently used will be needed again in the future.

  • Thus, Page which has lower ccount value should be kept in memory and page with more count value should be replaced.
